# 1C Form Designer

**Визуальный дизайнер управляемых форм 1С для Visual Studio Code**

[English](README.en.md)

---

## О проекте

1C Form Designer — расширение VS Code для визуального редактирования управляемых форм 1С:Предприятие. Работает напрямую с XML-файлами формата EDT-выгрузки, без необходимости запуска 1С или EDT.

### Преимущества

- **Кроссплатформенность** — Windows, macOS, Linux (везде, где работает VS Code)
- **Лёгкость** — менее 25 МБ (против ~2 ГБ у EDT)
- **Git-интеграция** — нативная работа с Git-воркфлоу
- **Round-trip** — изменения сериализуются обратно в XML без потери данных
- **Открытая архитектура** — расширяемость через API (в планах)

## Возможности (v0.1 MVP)

- Открытие и визуальное редактирование `Form.xml` (формат EDT)
- Дерево элементов формы с drag & drop
- Canvas — визуальное представление формы
- Инспектор свойств выбранного элемента
- Палитра элементов (Toolbox) — добавление новых элементов
- Undo/Redo с группировкой операций
- Три режима: Design, Structure, Source
- Сохранение неизвестных XML-блоков (Tier 3 preserve)
- Отслеживание внешних изменений файла

## Архитектура

Монорепо на базе Turborepo с четырьмя пакетами:

```
packages/
├── core-form/          # Ядро: FormModel, XML-парсер/сериализатор,
│                       # Layout Engine, Command Engine
├── shared/             # Общие типы: протокол сообщений, типы layout
├── webview-ui/         # UI: React + Zustand (дерево, canvas, инспектор)
└── vscode-extension/   # VS Code расширение: Custom Editor Provider
```

**FormModel** — единый источник истины. UI и расширение работают поверх модели, а не поверх XML.

## Быстрый старт

```bash
# Клонирование
git clone https://github.com/Desko77/1C-form-designer.git
cd 1c-form-designer

# Установка зависимостей
npm install

# Сборка всех пакетов
npm run build
```

### Запуск в режиме разработки

1. Откройте папку `packages/vscode-extension` в VS Code
2. Нажмите **F5** — запустится Extension Development Host
3. В Development Host откройте любой файл `Form.xml` из каталога `corpus/forms/`

### Сборка VSIX

```bash
cd packages/vscode-extension
npx @vscode/vsce package --no-dependencies
```

## Поддерживаемые элементы

Элементы форм поддерживаются на трёх уровнях:

| Уровень | Поддержка | Элементы |
|---------|-----------|----------|
| **Tier 1** — полная | Парсинг, рендер, редактирование | Группа, Страницы, Страница, Группа колонок, Командная панель, Поле ввода, Флажок, Поле надписи, Надпись, Кнопка, Таблица |
| **Tier 2** — базовая | Парсинг, базовый рендер | Переключатель, Текстовое поле, Число, Дата, Тумблер, Счётчик, Поле картинки, Декорация-картинка |
| **Tier 3** — сохранение | Сохранение XML без изменений | Все остальные типы |

- **TypeScript 5** — основной язык
- **Turborepo** — управление монорепо
- **React 18** + **Zustand** — UI
- **Vite** — сборка WebView

## Структура тестовых форм

Каталог `corpus/forms/` содержит тестовые XML-формы 1С:

```
corpus/forms/
├── simple/    # Простая форма (< 50 элементов)
├── medium/    # Средняя форма (50–200 элементов)
└── tables/    # Форма с таблицами
```
